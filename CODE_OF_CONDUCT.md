# Code of Conduct

## Our pledge

This is a single-author research project preparing for a science-fair
competition, built in the open under the MIT licence so others can
read it, learn from it, and adapt it. Anyone interacting with the
repository — opening an issue, sending a pull request, asking a
question — should expect a respectful, technical, and on-topic
exchange.

## Standards

**Encouraged**
- Technical, evidence-based feedback. "This claim isn't supported by
  the evidence" is welcome; "this paper is wrong" without engagement
  is not.
- Pointing out factual errors, citation gaps, or methodological issues.
  Science depends on it.
- Constructive disagreement about design choices.

**Not acceptable**
- Personal attacks, insults, or sustained derogatory commentary about
  contributors, the maintainer, the institution, or any group.
- Harassment, threats, or doxxing.
- Off-topic political or personal disputes.
- Spam or solicitation unrelated to the project.

## Scope

This Code of Conduct applies to all project spaces — issues, pull
requests, commit messages, code reviews — and to public communication
about the project (mentions on social media, conference Q&A, etc.).

## Enforcement

Reports go to **FerDa858@school.lu** with the subject
line `[VERA conduct] <summary>`. The maintainer will review every
report and respond within 7 days. Possible outcomes include a private
warning, a public correction, a temporary or permanent ban from the
repository, or no action if the report is deemed out of scope.

The maintainer reserves the right to close, edit, or remove
comments, commits, code, issues, or pull requests that violate this
Code of Conduct.
